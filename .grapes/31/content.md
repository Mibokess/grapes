## Goal

Add a `grapes next-id` subcommand that atomically reserves the next available issue ID across the main project and all worktrees. This prevents ID collisions when multiple worktree agents create issues concurrently.

## Context

Issue IDs are currently generated by agents using a shell one-liner:
```bash
next=$(( $(ls .grapes/ | sort -n | tail -1) + 1 ))
```

This only scans the local `.grapes/` directory. In worktrees (`.claude/worktrees/<name>/`), each agent has its own isolated `.grapes/` copy. Two agents creating issues simultaneously will compute the same next ID, causing collisions.

The Go codebase already has infrastructure for finding worktree `.grapes/` directories:
- `internal/data/loader.go:178` — `FindWorktreeIssuesDirs(projectRoot)` scans `.claude/worktrees/*/.grapes/`
- `internal/data/loader.go:172` — `ProjectRoot(issuesDir)` returns the parent of `.grapes/`
- `internal/data/loader.go:256` — `LoadAllSources()` merges issues from all sources

The main entrypoint is `main.go`, which already handles a `validate` subcommand (line 41).

### Finding the main project root from a worktree

From inside a worktree, `ProjectRoot()` returns the worktree root, not the main project. To find the main project root, use `git rev-parse --git-common-dir` which returns the main repo's `.git` directory — the parent of that is the main project root.

## Approach

### 1. Add `FindMainProjectRoot()` to `internal/data/loader.go`

Detects whether the current project root is inside a worktree and navigates to the real project root. Uses `git rev-parse --git-common-dir` or path-based detection (check for `.claude/worktrees/` in the path).

### 2. Add `NextID()` function to `internal/data/loader.go`

```go
func NextID(issuesDir string) (int, error)
```

- Finds the main project root
- Scans all `.grapes/` dirs (main + all worktrees) for numeric folder names
- Returns max(all IDs) + 1

### 3. Add lock file mechanism

Use a lock file at `<main-project-root>/.grapes/.lock` to serialize concurrent `next-id` calls:
- Acquire file lock (use `syscall.Flock` or `os.OpenFile` with `O_CREATE|O_EXCL`)
- Compute next ID
- Create the issue directory in the LOCAL `.grapes/`
- Release lock

This makes the scan-and-create atomic — no TOCTOU race.

### 4. Add `next-id` subcommand to `main.go`

```
grapes next-id
```

Output: just the numeric ID on stdout (e.g., `31`). The directory is already created.

### 5. Update skill instructions and README

Replace the shell one-liner in:
- `README.md` (line 86)
- `plugin/skills/grapes/SKILL.md` (if it references creation)
- `.agents/skills/grapes/SKILL.md`

New pattern:
```bash
next=$(grapes next-id)
```

## Acceptance Criteria

- [ ] `grapes next-id` prints a numeric ID to stdout and creates `.grapes/<id>/` locally
- [ ] The command scans all worktree `.grapes/` dirs, not just the local one
- [ ] Concurrent calls from different worktrees produce different IDs (lock file serialization)
- [ ] Works correctly from both the main project and from inside a worktree
- [ ] README and skill docs updated to use the new command
- [ ] Existing `validate` and TUI functionality unaffected
